WMWebServer 可以做什么？

WMWebServer 是基于C艹的静态HTTP网页服务器，她可以使用多线程非阻塞的方式响应请求。
WMWebServer 运行在 Windows 或 Linux 上。
WMWebServer 是闭源的免费软件，你可以将其用于各种非商业用途。
若要将 WMWebServer 用作商业用途，请联系作者。

启动参数

Windows:
WMWebServer.exe [-d] [-p:1217] [-r:X:/WMSystem/www]
Linux:
./wmwebserver.ucc [-d] [-p:1217] [-r:/data/www]
[-d] 可选。以调试模式运行服务器。当以调试模式运行服务器时，服务器将输出所有日志到屏幕。如果不指定，则仅输出警告和错误日志到屏幕。调试模式下服务器的速度可能减慢。
[-p:1217] 可选。在指定的端口上运行网站。如果不指定，则使用配置文件中的值。Linux中非root用户不能绑定1024以下的端口。
[-r:X:/WMSystem/www] 可选。使用指定的目录作为网站根目录。如果不指定，则使用配置文件中的值。请确认应用程序对该目录有访问权限。

若没有指定对应参数，WMWebServer 将使用配置文件中的值。

配置文件

WMWebServer 使用 config.wmst 作为配置文件。

对于 Cured 版本的 WMWebServer，该文件位于 /etc/WMWebServer/ 或 X:/WMSystem/etc/WMWebServer/，其中X为计算机上系统磁盘的盘符。

在 Windows 上，该文件的默认内容如下:
!#port=1217
!#path=X:/WMSystem/www
其中，X为计算机上系统磁盘的盘符。

在 Linux 上，该文件的默认内容如下:
!#port=1217
!#path=/data/www
对于 Portable 版本的 WMWebServer，该文件位于应用程序相同目录下。

该文件的默认内容如下:
!#port=1217
!#path=PATH/www
其中，PATH为应用程序所在的目录。

若无法找到配置文件，WMWebServer 会使用其默认值。

特性

WMWebServer 的默认索引为 index.html。
WMWebServer 不会自动重定向。例如，当访问 /act 时，WMWebServer 不会将请求重定向到 /act/，而是返回错误页面。
当找不到网站目录下的 index.html 时，WMWebServer 可能会返回标题为 200 OK 的页面。
当请求包含参数时，WMWebServer 不会处理任何参数，她将直接返回页面。
如果发送了 POST 请求，老版本 WMWebServer 返回空内容或返回 500 Internal Server Error，新版本 WMWebServer 会将其作为 GET 请求处理。
WMWebServer Windows 版本的线程数量不是受限的，当有大量突发请求入站时，WMWebServer Windows 可能会大量占用 CPU 导致系统卡顿；WMWebServer Linux 版本默认限制最大32个线程。
